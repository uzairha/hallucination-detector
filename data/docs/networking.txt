TCP, the Transmission Control Protocol, is a connection-oriented protocol that guarantees reliable, ordered delivery of data between two hosts. Before data is transmitted, TCP performs a three-way handshake to establish a connection. It uses acknowledgments, retransmissions, and flow control to ensure every packet arrives correctly and in order. TCP is used by applications where data integrity is critical, such as web browsing, email, and file transfer.

UDP, the User Datagram Protocol, is a connectionless protocol that sends packets without establishing a connection or guaranteeing delivery. There is no handshake, no acknowledgment, and no retransmission. UDP has lower latency and overhead than TCP, making it suitable for applications where speed matters more than reliability, such as video streaming, online gaming, and DNS lookups.

Data transmission over a network is divided into layers described by models such as the OSI model. The transport layer, where TCP and UDP operate, is responsible for end-to-end communication between processes. Below it, the network layer handles routing packets across networks using IP addresses. The physical and data link layers handle the actual transmission of bits over physical media.
